# 符印守卫（快速落地版策划）

## 项目一句话

玩家站在摄像头前，通过手势施放不同法术，抵御沿三条路线来袭的敌人。

## 当前开发目标

- 将现有“离散指令接球”MVP 迭代为“体感施法防守”MVP
- 保留浏览器 + MediaPipe Hands + Canvas 的技术路线
- 优先做出“像真的在施法”的感觉，而不是堆复杂系统

## 核心设计目标

1. 玩家能明显感受到自己在施法，而不是在替代键盘输入。
2. 体感输入必须低负担，不需要持续举手。
3. 首个可玩版本要可展示、可解释、可快速验证。
4. 玩法结构要能在后续继续扩展为更完整的体感游戏系统。

## 核心玩法循环

1. 敌人沿左 / 中 / 右三条路线向防线推进。
2. 玩家观察敌人类型与所在路线。
3. 玩家单手入镜，系统识别当前候选法术。
4. 手在同一区域稳定停留约 0.5 秒后确认施法。
5. 法术释放，对目标产生击碎、冻结或阻挡效果。
6. 成功防御得分；漏防则扣除生命或记一次失误。
7. 敌人刷新速度逐步提升，玩家持续应对。

## 输入机制

### 输入原则

- 单手输入
- 离散施法
- 稳定停留确认
- 一次入镜只确认一次施法

### 输入映射

- 左区：冰霜术
- 中区：护盾术
- 右区：火焰术

### 确认规则

- 手入镜后先进入候选态，不立即触发
- 在同一区域稳定停留约 0.5 秒才确认
- 中途离镜则取消
- 中途换区则重新计时
- 已确认后，本次入镜不重复触发
- 手离镜后，系统重新允许下一次施法

## 法术系统

### 火焰术

- 输入：右区
- 定位：直接消灭目标
- 主要用途：处理普通敌人

### 冰霜术

- 输入：左区
- 定位：冻结或减速目标
- 主要用途：处理高速敌人、争取反应时间

### 护盾术

- 输入：中区
- 定位：防御 / 容错
- 主要用途：阻挡一次命中或守住漏网目标

## 敌人系统

### 普通魔球

- 中速推进
- 适合使用火焰术处理

### 疾行碎影

- 速度更快
- 更适合使用冰霜术处理

### 重压核心

- 威胁更高，容错更低
- 适合使用护盾术兜底

## 战场结构

- 三条清晰可读的来袭路线
- 敌人自上而下推进
- 底部为玩家防线 / 结界核心
- HUD 展示生命、分数、当前候选法术、识别状态等信息

## 节奏设计

### 单局目标

- 1~3 分钟内完成一局体验

### 节奏阶段

1. 0~20 秒：只出现普通敌人，完成输入教学
2. 20~60 秒：混入高速敌人，要求玩家区分法术用途
3. 60 秒后：更多混合敌人与更高刷新频率，形成压迫感

## 反馈设计

### 输入反馈

- 手入镜时显示识别光圈
- 当前候选法术区域高亮
- 确认过程中显示聚能 / 蓄力进度
- 确认成功时显示短暂法阵闪光

### 法术反馈

- 火焰：爆裂、火焰轨迹、暖色冲击
- 冰霜：冻结扩散、蓝白色冷光
- 护盾：结界展开、屏障波纹

### 命中反馈

- 击碎 / 冻结 / 阻挡反馈明显
- 分数跳字
- 连续成功时可加入连击表现

## MVP 范围锁定

### 必做

- 三路线
- 三法术
- 三类敌人
- 稳定停留确认
- 分数 / 生命（或失误次数）/ 失败 / 重开
- 最基础的施法视觉反馈

### 暂不做

- 双手识别
- 复杂手型分类
- 画符轨迹
- 剧情与成长系统
- 多人模式

## 推荐落地顺序

1. 将接球玩法替换为三路线防守结构
2. 将左 / 中 / 右移动语义替换为三法术语义
3. 为三类敌人补上最小差异化行为
4. 接入稳定停留确认逻辑
5. 补候选高亮、聚能、命中效果
6. 调整节奏与分数，使其适合答辩演示

## 验收标准

1. 玩家第一次看画面就知道自己是在“施法”。
2. 玩家能区分三种法术的作用。
3. 玩家不需要持续举手。
4. 一局内能出现明确的判断与反馈，而不是机械重复操作。
5. 外部观众能在短时间内理解交互亮点与系统价值。
