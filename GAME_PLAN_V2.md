# 符印守卫（新版策划案）

## 1. 项目一句话

玩家无需键盘和鼠标，仅通过摄像头前的手势完成菜单操作、设置调整与战斗施法，在“结印”式交互中守住结界。

## 2. 本次改版目标

- 从“手在画面哪个位置”升级为“手做出了什么手势”
- 从“局内有手势”升级为“全流程无键鼠手势交互”
- 强化施法幻想，让玩家像结印一样释放法术
- 保持毕设原型可落地，优先做少量稳定手势与完整闭环

## 3. 核心设计方向

新版《符印守卫》的重点不再是空间点选，而是**语义化手势输入**。

玩家的手势既是：

- 界面操作方式
- 战斗输入方式
- 世界观表达方式

项目的核心价值是证明：**手势识别不只是技术展示，也可以成为完整可玩的游戏交互系统。**

## 4. 核心体验目标

1. 玩家从进入主界面开始就不需要键鼠。
2. 玩家能明显感觉自己是在“结印施法”，而不是在空中点按钮。
3. 手势数量少而清晰，识别稳定，适合首次体验。
4. 一局体验控制在 2~5 分钟，适合毕设答辩演示。

## 5. 全手势交互闭环

### 5.1 系统层交互

用于：

- 进入主菜单
- 选择开始游戏
- 打开设置
- 调整参数
- 教程查看
- 局后重开或返回主菜单

### 5.2 玩法层交互

用于：

- 识别法术手势
- 进行结印确认
- 触发不同法术效果
- 在战斗压力下切换手势应对敌人

## 6. UI 手势方案（MVP）

为保证稳定性，菜单手势保持极简，只做 3 个核心动作：

### 6.1 指向手势

- 作用：移动界面焦点 / 选择按钮
- 建议形式：食指伸出，其余手指收拢

### 6.2 停留确认

- 作用：确认当前焦点项
- 规则：指向目标后稳定停留 0.8 秒

### 6.3 返回手势

- 作用：返回上一级或取消当前操作
- 建议形式：向左挥手，或张掌后短暂停留

## 7. 菜单流程

### 7.1 主菜单

包含：

- 开始游戏
- 设置
- 教程
- 退出演示（可选）

流程：

1. 玩家入镜
2. 系统检测到手部，激活主界面
3. 玩家用指向手势移动焦点
4. 停留确认进入对应页面

### 7.2 设置界面

建议保留少量必要选项：

- 手势确认时长
- 识别灵敏度
- 音量开关
- 难度等级

调整逻辑：

- 指向某个设置项
- 左右滑手切换参数值
- 停留确认保存
- 返回手势离开页面

### 7.3 局后界面

显示：

- 得分
- 存活时间
- 施法次数
- 命中率

可选操作：

- 再来一局
- 返回主菜单

## 8. 结印施法系统

## 8.1 设计原则

- 先做静态手势，不做复杂轨迹
- 先做 3 个核心法术，避免范围失控
- 所有法术都需要“候选识别 + 稳定确认”
- 让每个手势都具有明确视觉识别差异

## 8.2 手势词典（MVP）

| 法术 | 建议手势 | 功能定位 | 说明 |
|---|---|---|---|
| 火焰印 | 握拳 | 直接攻击 | 力量感强，适合做主要输出 |
| 冰霜印 | 双指 / V 手势 | 控制减速 | 识别差异明显，适合功能型法术 |
| 护盾印 | 张开手掌 | 防御展开 | 与“展开屏障”语义一致 |

## 8.3 施法流程

1. 玩家观察敌人来袭
2. 做出对应结印手势
3. 系统进入候选识别状态
4. 玩家保持手势约 0.4~0.8 秒
5. 法阵完成，法术释放
6. 法术进入短暂冷却或结束反馈

## 9. 战斗玩法

玩家需要在敌人逼近结界前，使用正确的结印进行防守。

### 核心循环

1. 敌人刷新并推进
2. 玩家识别威胁类型
3. 玩家结印施法
4. 法术产生攻击、控制或防御效果
5. 成功防守得分，漏防则结界受损
6. 敌人节奏逐步加快，形成压力

## 10. 敌人与法术关系

建议保留 3 类敌人，方便快速建立策略关系：

- 普通敌人：优先用火焰印清除
- 高速敌人：优先用冰霜印减速或冻结
- 高威胁敌人：需要护盾印进行容错防守

这样玩家会形成明确认知：

**不是随便做个动作，而是在局势下结出正确的印。**

## 11. 反馈设计

### 11.1 菜单反馈

- 当前焦点高亮
- 停留确认环
- 返回动作动画提示

### 11.2 手势识别反馈

- 当前识别到的手势名称
- “识别中 / 已锁定 / 施法成功”状态提示
- 识别失败时给出重新摆手提示

### 11.3 施法反馈

- 火焰：爆裂、暖色冲击
- 冰霜：冻结扩散、冷色粒子
- 护盾：结界展开、波纹屏障

## 12. MVP 范围锁定

### 必做

- 主菜单手势导航
- 设置界面基础手势操作
- 3 种稳定可识别的施法手势
- 候选识别与稳定确认机制
- 一套完整战斗闭环
- 局后手势重开 / 返回主菜单

### 暂不做

- 双手复杂结印
- 连续画符轨迹
- 过多法术分支
- 成长养成系统
- 多人模式

## 13. 风险与应对

### 风险 1：误识别

- 原因：光照、镜头角度、玩家手型差异
- 应对：减少手势数量，强化候选态与确认时间

### 风险 2：学习成本高

- 原因：菜单和战斗都用手势，首次理解有门槛
- 应对：加入简短教程页，并始终显示可用手势提示

### 风险 3：开发范围过大

- 原因：全手势交互比原方案复杂
- 应对：先完成“菜单三手势 + 战斗三印式”的最小闭环

## 14. 毕设展示价值

这版方案相较旧版更适合毕业设计展示，原因在于：

1. **交互完整**：从菜单到战斗都由手势驱动
2. **表达更统一**：手势既是操作，也是世界观的一部分
3. **展示更直观**：评审能一眼看出项目亮点
4. **研究价值更明确**：重点变成“手势如何构成完整游戏交互”

## 15. 当前推荐落地顺序

1. 先做主菜单手势导航
2. 再做设置页的基础手势交互
3. 将战斗输入切换为 3 个结印手势
4. 补充候选态、确认态与施法反馈
5. 最后加入局后结算与手势重开

---

这份新版策划案的目标不是一步到位做成完整商业游戏，而是完成一个**全手势、可演示、可答辩、可继续迭代**的体感游戏原型。
